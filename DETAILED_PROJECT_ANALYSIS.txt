U1) VOICE FLOW LOGIC (CONTINUOUS UNTIL VARIANTS, THEN AUTO-STOP)
--------------------------------------------------------------------------------
File:
- src/app/(dashboard)/dashboard/generate/page.tsx

Previous behavior observed:
- Voice mode either auto-looped indefinitely or required manual restart every
  turn depending on interim patch state.

Current implemented behavior:
- While the conversation is still in information-gathering phase (assistant is
  asking follow-up questions, no ad variants returned), voice resumes
  automatically after TTS playback.
- Once the assistant returns final ad variants (ads array present), the voice
  recorder is stopped and state returns to idle.
- After variants are shown, the assistant does NOT keep listening to ambient
  room audio and does NOT continue replying unexpectedly.

Practical UX outcome:
- Hands-free multi-turn discovery conversation.
- Safe, explicit stop point at "3 variants delivered" stage.

0) EXECUTIVE SNAPSHOT
--------------------------------------------------------------------------------
AdGen is a full-stack AI ad generation platform built as a single Next.js 14
application (App Router) with both UI and backend APIs in one codebase.

The product focus is:
- Conversational ad copy generation
- Voice-first interaction (speech input + spoken responses)
- AI poster generation
- AI video commercial generation
- Campaign and ad management dashboard
- Public gallery and feedback loop
- Sentiment + emotion analysis for ad feedback

Tech stack is modern TypeScript-first, with Prisma + SQLite for persistence,
NextAuth for authentication, OpenAI + Replicate for AI features, and FFmpeg for
video post-processing.

Observed codebase scope:
- ~92 files in src + prisma
- ~25 files under src/app/api (API surface)

7) API SURFACE - ENDPOINT BY ENDPOINT ANALYSIS
--------------------------------------------------------------------------------
7.1 AI conversation and generation
- POST /api/chat
  - Validates API key availability
  - Rate limiting category: chat
  - Sanitizes user messages for injection/XSS patterns
  - Sends system + history to GPT model
  - Parses `json:ads` fenced block if returned
  - Persists ads + conversation/messages for authenticated users

- POST /api/generate-image
  - Auth required
  - Rate limiting category: image
  - Builds prompt from ad + customization context
  - Calls image model
  - Saves base64 output to public/generated-images
  - Updates ad.imageUrl if adId provided

- POST /api/generate-video
  - Auth required
  - Requires OpenAI and Replicate keys
  - Rate limiting category: video
  - Generates 4-scene commercial script
  - Creates scene predictions sequentially with retry/backoff
  - Initializes job state file in public/generated-videos/jobs/<jobId>
  - Triggers background TTS generation
  - Marks ad videoStatus=processing and stores replicateId as jobId

- GET /api/generate-video/status/[jobId]
  - Auth required
  - Poll endpoint for multi-stage job
  - Downloads completed scene clips
  - On all clip success: merges with FFmpeg pipeline
  - Handles fallback when FFmpeg unavailable/failure
  - Updates ad videoUrl and videoStatus

- GET /api/generate-video/[predictionId]
  - Legacy/simpler prediction status endpoint
  - Polls one prediction and stores final video

7.2 Voice endpoints
- POST /api/transcribe
  - Accepts form-data audio file
  - Uses whisper transcription

- POST /api/tts
  - Accepts text JSON
  - Returns audio/mpeg binary response

7.3 Ads and engagement
- GET /api/ads
  - Auth required
  - Returns user ads with campaign name
  - Parses hashtags JSON string to array

- POST /api/ads
  - Auth required
  - Plan limit check based on user.plan (free vs pro)
  - Creates ad with required fields

- PATCH/DELETE /api/ads/[id]
  - Auth + ownership checks
  - Partial update for multiple ad fields
  - Delete ad by owner

- POST /api/ads/engagement
  - Auth required
  - Validates engagement type: impression/click/like/share
  - Increments corresponding counters

- GET/POST /api/ads/ab-test
  - POST creates variant B from AI prompt using an existing ad
  - GET returns A/B data and computed rates (CTR, engagementRate)

7.4 Campaigns
- GET/POST /api/campaigns
  - Auth required
  - List campaigns with ad counts
  - Create campaign with plan cap enforcement

- GET/PATCH/DELETE /api/campaigns/[id]
  - Auth + ownership checks
  - Full lifecycle operations

7.5 Conversations
- GET/POST /api/conversations
  - List user conversations + metadata
  - Create conversation with optional initial messages

- GET/PATCH/DELETE /api/conversations/[id]
  - Fetch full conversation + parsed ads per message
  - Update title/add message
  - Delete conversation

7.6 Feedback and gallery
- GET/POST /api/feedback
  - Auth required for private feedback management
  - Uses sentiment analysis utility
  - GET supports ad-specific or aggregate summary mode

- GET /api/gallery
  - Public endpoint
  - Returns only published ads with author + recent feedback

- POST /api/gallery/[id]
  - Public interaction endpoint
  - Supports action=like and action=feedback
  - Feedback also runs sentiment pipeline

7.7 User and helper APIs
- GET/PATCH /api/user
  - Profile retrieval + profile/password updates

- POST /api/help
  - Help bot endpoint with dedicated system prompt

- POST /api/translate
  - Auth required
  - Supports both field-object translation and plain text translation

- POST /api/poster
  - Returns AI-generated design spec JSON for poster concepts

- GET /api/auth/[...nextauth]
  - NextAuth route binding


8) CHAT AND VOICE UX WORKFLOW (MINUTE INTERACTION DETAILS)

8.1 Chat message lifecycle

- Silence detection:
  - marks speech seen when above threshold
  - auto-stop after timeout once silence persists
- Blob sent to /api/transcribe
- On transcript:
  - invoke chat flow
  - optional TTS playback
  - auto-resume listening for conversational loop

8.3 Chat ad card utilities

- Trigger A/B variant generation

9) VIDEO GENERATION PIPELINE - INTERNAL STAGES
--------------------------------------------------------------------------------
9.1 Script generation
- AI builds 4 scenes with:
  - visual prompt per scene
  - scene-level voiceover
  - overlay text + position
  - overall brand mood + full voiceover script

9.2 Scene rendering
- Predictions launched sequentially with stagger delay to avoid provider limits
- Retry logic handles 429-like rate limit errors

9.3 Job state persistence
- Job metadata in JSON file tracks:
  - prediction states
  - user ownership
  - merge status
  - final URL
  - errors
  - TTS readiness

9.4 Merge phase
- Download clip outputs
- Concat clips and normalize dimensions/fps
- Draw text overlays by scene timing
- Generate ambient music by mood (or use custom file if present)
- Mix voice + bgm when available
- Export final MP4 and expose under public/generated-videos

9.5 Fallback behavior
- If FFmpeg unavailable or merge fails:
  - returns first clip as fallback output
  - still marks generation succeeded with fallback metadata

14) KNOWN IMPLEMENTATION CHARACTERISTICS AND LIMITATIONS
--------------------------------------------------------------------------------
14.1 Architecture boundaries
- Single app architecture simplifies development and deployment for small/medium
  scale usage.
- API and UI tightly coupled in one codebase.

14.2 Rate limiter scope
- In-memory limiter does not persist across instances; for multi-instance
  production a shared store (Redis) would be needed.

14.3 Asset storage model
- Generated media stored on local filesystem under public/.
- This is simple for single-instance hosting, but object storage is preferred
  for robust production scaling.

14.4 A/B logic simplification
- Variant creation currently generates one alternate variant per request with
  JSON parse heuristics from model output.

14.5 Miscellaneous technical notes
- There is both a job-based video status system and a predictionId endpoint,
  indicating some backward compatibility/transition in video architecture.
- ui-store default theme is dark while root layout includes className="dark".
