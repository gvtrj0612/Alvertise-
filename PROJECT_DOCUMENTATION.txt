================================================================================
1. PROJECT OVERVIEW
================================================================================

AdGen is a full-stack AI-powered advertisement generation platform built with
Next.js 14. It enables users to create high-converting ad copy, poster images,
and promotional videos through a conversational AI interface with voice
interaction support. The system leverages multiple AI models (OpenAI GPT-4o-mini,
DALL-E 3, Whisper, TTS-1, and Replicate Stable Video Diffusion) to deliver an
end-to-end ad creation workflow.

The platform supports multi-language ad generation (English, Hindi, Kannada,
Spanish, French), multiple advertising platforms (Facebook, Instagram, Google
Ads, LinkedIn, Twitter/X), and includes built-in security features like prompt
injection detection, XSS protection, and IP-based rate limiting.


================================================================================
2. UNIQUE SELLING PROPOSITIONS (USP)
================================================================================

1. CONVERSATIONAL AD CREATION
   - Unlike form-based ad tools, AdGen uses a natural chat interface powered by
     GPT-4o-mini. Users describe their product or service in conversation, and
     the AI asks follow-up questions one at a time to gather context before
     generating ads. This makes ad creation feel intuitive and accessible.

2. VOICE-FIRST INTERACTION
   - Full voice workflow: speak your ad brief via microphone (transcribed by
     Whisper), get AI responses read aloud via TTS-1. Includes automatic silence
     detection using Web Audio API AnalyserNode for hands-free operation.

3. COMPLETE AD PIPELINE (Text -> Image -> Video)
   - Generate ad copy, then create a poster image with DALL-E 3, then animate
     it into a short video with Stable Video Diffusion - all from a single
     conversation. No need to switch between different tools.

4. MULTI-LANGUAGE SUPPORT
   - Generate ads in 5 languages: English, Hindi, Kannada, Spanish, French.
     The AI adapts cultural nuances for each language.

5. PLATFORM-SPECIFIC OPTIMIZATION
   - Ads are tailored to each platform's character limits and best practices:
     Facebook (headline <= 40 chars), Google Ads (headline <= 30 chars),
     Instagram (hashtag-heavy, emoji-friendly), LinkedIn (professional tone),
     Twitter/X (concise, <= 280 chars).

6. PROFESSIONAL COPYWRITING FRAMEWORKS
   - Uses industry-standard frameworks: AIDA (Attention-Interest-Desire-Action),
     PAS (Problem-Agitate-Solution), BAB (Before-After-Bridge), 4U (Useful,
     Urgent, Unique, Ultra-specific). Each generation produces 3 variations
     using different frameworks.

7. BUILT-IN SECURITY
   - Prompt injection detection (16 regex patterns), XSS protection (6 HTML/
     script patterns), input sanitization, message length limits, and IP-based
     rate limiting on all endpoints.

------------------------------------------------------------------------------
3.1 CHAT-BASED AD GENERATION
------------------------------------------------------------------------------
Technology: OpenAI GPT-4o-mini
Route:      /api/chat (POST)
Component:  src/components/chat/chat-window.tsx
Store:      src/store/chat-store.ts (Zustand)

How it works:
- User sends a message describing their product/service
- The AI asks follow-up questions (one at a time) to gather:
  * Product/service name and description
  * Target audience (demographics, psychographics, pain points)
  * Advertising platform
  * Desired tone
  * Key selling points / USP
  * Call-to-action
  * Budget context / promotional offers
- Once enough context is gathered, the AI generates 3 ad variations
- Each variation uses a different copywriting framework (AIDA, PAS, BAB)
- Ads are returned as structured JSON within the response
- Ads are automatically saved to the SQLite database via Prisma

Parameters:
  Model:       gpt-4o-mini
  Temperature: 0.7
  Max Tokens:  1500

Output format:
  Each ad contains: headline, primaryText, description, callToAction,
  platform, hashtags[]

------------------------------------------------------------------------------
3.2 VOICE INTERACTION (Speech-to-Text + Text-to-Speech)
------------------------------------------------------------------------------
Technologies: OpenAI Whisper (STT), OpenAI TTS-1 (TTS), Web Audio API

Speech-to-Text:
  Route:     /api/transcribe (POST)
  Model:     whisper-1
  Format:    Accepts audio File via FormData
  Hook:      src/hooks/use-voice-recorder.ts

  How it works:
  - User clicks mic button to start recording
  - MediaRecorder captures audio from getUserMedia()
  - Web Audio API AnalyserNode monitors audio levels in real-time
  - Automatic silence detection: if audio RMS drops below threshold (0.01)
    for 2 seconds, recording stops automatically
  - Audio blob sent to /api/transcribe, which forwards to Whisper
  - Transcribed text is injected into the chat input

Text-to-Speech:
  Route:     /api/tts (POST)
  Model:     tts-1
  Voice:     nova
  Hook:      src/hooks/use-audio-player.ts

  How it works:
  - After AI generates a response, the text content is sent to /api/tts
  - TTS-1 returns an audio/mpeg buffer
  - Frontend creates an AudioContext, decodes the buffer, and plays it
  - Audio state is tracked in the Zustand store (isPlaying, audioRef)

------------------------------------------------------------------------------
3.3 AI POSTER IMAGE GENERATION
------------------------------------------------------------------------------
Technology: OpenAI DALL-E 3
Route:      /api/generate-image (POST)
Component:  src/components/chat/ad-preview-card.tsx

How it works:
- Each generated ad card has a "Generate Poster" button
- When clicked, the ad's headline, primary text, CTA, platform, theme,
  colors, audience, and language are sent to /api/generate-image
- The API constructs a detailed DALL-E 3 prompt incorporating all parameters
- DALL-E 3 generates a 1024x1024 image in b64_json format
- The image is saved locally to /public/generated-images/ directory
- The image URL is stored in the Ad record in the database
- The poster is displayed in the ad preview card

Parameters:
  Model:           dall-e-3
  Size:            1024x1024
  Response Format: b64_json
  N:               1

------------------------------------------------------------------------------
3.4 AI VIDEO GENERATION
------------------------------------------------------------------------------
Technology: Replicate (Stable Video Diffusion)
Routes:     /api/generate-video (POST), /api/generate-video/[predictionId] (GET)
Component:  src/components/chat/video-preview.tsx

How it works:
- Step 1: If no source image exists, generate one via DALL-E 3
- Step 2: Convert image to base64 data URI
- Step 3: Send to Replicate's Stable Video Diffusion model
  * Model version: d68b16aea809e0dda306862ab5e22e6eb5809e466b66c76a1a9e5a0e3e5b7fb3
  * Parameters: motion_bucket_id=127, fps=7, cond_aug=0.02
- Step 4: Replicate returns a prediction ID (video renders asynchronously)
- Step 5: Frontend polls /api/generate-video/[predictionId] every 5 seconds
- Step 6: When status = "succeeded", the video URL is returned
- Step 7: Video is downloaded and saved locally to /public/generated-videos/
- Step 8: Ad record updated with videoUrl and videoStatus in database

Video Preview Component (video-preview.tsx):
- Shows step-by-step progress indicators:
  1. Generate Image (DALL-E 3)
  2. Render Video (Replicate SVD)
  3. Download & Save
- Progress bar with percentage
- Source image preview during rendering
- Auto-plays video on completion
- Error state with retry button
- Maximum 60 polls (5 minutes timeout)

8. SECURITY FEATURES
================================================================================

------------------------------------------------------------------------------
8.1 PROMPT INJECTION DETECTION
------------------------------------------------------------------------------
File: src/lib/sanitize.ts

16 regex patterns detect common injection attempts:
  - "ignore previous instructions"
  - "disregard previous"
  - "forget your instructions"
  - "you are now a [non-ad role]"
  - "system:" prefix injection
  - "[INST]" and "<<SYS>>" token injection
  - "act as [non-marketing role]"
  - "pretend you're [non-ad role]"
  - "override your system/prompt"
  - "reveal your system/prompt"
  - "what are your instructions"
  - "DAN" (Do Anything Now)
  - "do anything now"
  - "jailbreak"

When detected: threats are logged with IP address, input is still sanitized
and processed (does not block the request, but staff can monitor).

------------------------------------------------------------------------------
8.2 XSS PROTECTION
------------------------------------------------------------------------------
6 HTML/script patterns:
  - <script> tags
  - javascript: protocol
  - Event handler attributes (onerror, onload, onclick, onmouseover)
  - <iframe>, <object>, <embed> tags

All HTML tags are stripped from user input. Angle brackets are removed.

------------------------------------------------------------------------------
8.3 RATE LIMITING
------------------------------------------------------------------------------
File: src/lib/rate-limit.ts

In-memory Map-based rate limiting per IP address:
  - Chat:     20 requests / 1 minute
  - TTS:      30 requests / 1 minute
  - Image:    5 requests / 1 minute
  - Video:    3 requests / 5 minutes
  - Signup:   5 requests / 15 minutes
  - Default:  60 requests / 1 minute

Returns HTTP 429 with Retry-After header when exceeded.
Expired entries cleaned every 5 minutes.

------------------------------------------------------------------------------
8.4 API KEY VALIDATION
------------------------------------------------------------------------------
All 5 AI-powered routes validate that API keys are configured before
processing requests. Returns HTTP 503 with a helpful error message if
keys are missing or still set to placeholder values.

------------------------------------------------------------------------------
8.5 AUTHENTICATION MIDDLEWARE
------------------------------------------------------------------------------
File: src/middleware.ts

All /dashboard/* routes are protected. Unauthenticated users are redirected
to /login. Uses NextAuth v5 JWT session validation.

------------------------------------------------------------------------------
8.6 INPUT LENGTH LIMITS
------------------------------------------------------------------------------
User messages are truncated to 5,000 characters maximum to prevent
abuse and excessive API costs.

9. APPLICATION FLOWS
================================================================================

------------------------------------------------------------------------------
9.1 AUTHENTICATION FLOW
------------------------------------------------------------------------------

New User Registration:
  1. User visits /signup
  2. Enters name, email, password
  3. POST /api/signup -> bcrypt hash password -> Prisma create User
  4. Redirected to /login
  5. Signs in with credentials
  6. NextAuth creates JWT session
  7. Redirected to /dashboard

Google OAuth:
  1. User clicks "Sign in with Google" on /login
  2. Redirected to Google consent screen
  3. Google returns auth code to callback URL
  4. NextAuth exchanges code for tokens, gets user profile
  5. Prisma upserts User record (creates if new)
  6. JWT session created
  7. Redirected to /dashboard

------------------------------------------------------------------------------
9.2 AD GENERATION FLOW (Chat)
------------------------------------------------------------------------------

  1. User opens /dashboard -> Chat Window is the main interface
  2. User types or speaks a message about their product
  3. Message goes through sanitization (injection detection, XSS stripping)
  4. Customization panel values are attached if configured
  5. POST /api/chat with messages array + customization object
  6. Server builds system prompt (base prompt + customization context)
  7. OpenAI GPT-4o-mini processes the conversation
  8. Response is parsed for ```json:ads blocks
  9. If ads found: JSON parsed, ads saved to DB, conversation persisted
  10. Response returned: { content, ads[], conversationId }
  11. Frontend renders message bubble + ad preview cards
  12. If voice mode active: text content sent to TTS for audio playback

------------------------------------------------------------------------------
9.3 VOICE INTERACTION FLOW
------------------------------------------------------------------------------

  Speaking to AI:
  1. User clicks microphone button
  2. Browser requests microphone permission (getUserMedia)
  3. MediaRecorder starts capturing audio
  4. AudioContext + AnalyserNode monitors audio levels
  5. requestAnimationFrame loop checks RMS audio level
  6. If RMS < 0.01 for 2+ seconds -> auto-stop recording
  7. Audio blob created from recorded chunks
  8. POST /api/transcribe with FormData containing audio file
  9. Whisper transcribes audio to text
  10. Transcribed text placed in chat input
  11. User can edit or send immediately

  AI Speaking Back:
  1. AI response text received from /api/chat
  2. Text sent to POST /api/tts
  3. TTS-1 (nova voice) generates audio/mpeg buffer
  4. Frontend creates AudioContext, decodes ArrayBuffer
  5. Audio played through speakers
  6. Play/pause state tracked in Zustand store

------------------------------------------------------------------------------
9.4 POSTER IMAGE GENERATION FLOW
------------------------------------------------------------------------------

  1. Ad preview card displayed in chat
  2. User clicks "Generate Poster" button
  3. POST /api/generate-image with ad details + customization
  4. Server constructs DALL-E 3 prompt from all parameters
  5. DALL-E 3 generates 1024x1024 image (b64_json format)
  6. Image decoded from base64, saved to /public/generated-images/
  7. Ad record updated with imageUrl in database
  8. Response: { imageUrl } -> frontend displays the poster
  9. Image appears in the ad card and on the /dashboard/ads page

------------------------------------------------------------------------------
9.5 VIDEO GENERATION FLOW
------------------------------------------------------------------------------

  1. User clicks "Generate Video" on an ad card
  2. VideoPreview component enters step-by-step generation mode

  Step 1 - Source Image:
  3. If ad already has a poster image -> use it as source
  4. If no image -> POST /api/generate-video triggers DALL-E 3 generation
  5. Image converted to base64 data URI

  Step 2 - Video Rendering:
  6. Base64 image sent to Replicate Stable Video Diffusion
  7. Replicate returns prediction ID (async processing)
  8. Ad record updated: videoStatus="processing", replicateId stored
  9. Frontend starts polling GET /api/generate-video/[predictionId]
  10. Poll every 5 seconds, display progress bar
  11. Maximum 60 polls (5-minute timeout)

  Step 3 - Download & Save:
  12. When Replicate status = "succeeded", video URL returned
  13. Video downloaded from Replicate CDN
  14. Saved locally to /public/generated-videos/
  15. Ad record updated: videoUrl set, videoStatus="succeeded"
  16. Video auto-plays in the preview component

  Error Handling:
  - If Replicate status = "failed" -> show error with retry button
  - If polling times out -> show timeout error with retry
  - Network errors -> show error state with retry option

14. SETUP & INSTALLATION GUIDE
================================================================================

Prerequisites:
  - Node.js 18+ installed
  - npm or yarn package manager
  - OpenAI API key (https://platform.openai.com)
  - Replicate API token (https://replicate.com)
  - Google OAuth credentials (optional, for Google sign-in)

Steps:

  1. Clone or navigate to the project directory
     cd AdGen/frontend

  2. Install dependencies
     npm install

  3. Set up environment variables
     - Copy .env.example to .env.local (or create .env.local)
     - Add your API keys (see Section 11 above)

  4. Initialize the database
     npx prisma generate
     npx prisma db push

  5. Run the development server
     npm run dev

  6. Open the application
     http://localhost:3000

  7. Create an account
     - Go to /signup to create a new account
     - Or use Google OAuth on the /login page

  8. Start generating ads
     - Go to /dashboard
     - Type or speak your ad brief in the chat
     - Configure the customization panel
     - Generate ad copy, posters, and videos
